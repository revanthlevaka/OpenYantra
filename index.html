01 / The Problem
01
Every session starts blank
Your AI has no memory of your projects, decisions, preferences, or unfinished threads. You repeat the same context every single session.
02
Work disappears in compaction
Long conversations get compressed. Active decisions, mid-task context, and unresolved threads vanish without warning during long sessions.
03
Memory you cannot inspect
Cloud memory systems are black boxes. You cannot see what was recorded, verify it is accurate, or correct what is wrong.
04
Agents overwriting each other
In multi-agent workflows, agents write to the same memory store without coordination. Race conditions, silent overwrites, and data corruption follow.
05
OpenYantra solves all of this
One file. One writer. Every write signed and audited. Your context injected automatically at session start. Unfinished work persists across compaction.
06
A file you own and can edit
Your Chitrapat is a `.ods` spreadsheet. Open it in LibreOffice. Read every record. Edit anything. Your changes always override agent writes.
02 / How It Works

One file. One trusted writer. Complete control.

Only Chitragupta (LedgerAgent) writes to your memory file. All other agents are read-only. Every write is sealed with SHA-256 and permanently recorded in the Agrasandhanī audit trail.

06 / Inspired by Chitragupta

3,000 years of mythology.
Arrived at the same architecture.

Chitragupta (चित्रगुप्त) is the Hindu God of Data -- the divine scribe who maintains a complete, incorruptible record of every human soul's deeds in the Agrasandhanī. The name means "the hidden picture" -- a complete record that persists invisibly, always accurate, always available.

The architecture of OpenYantra was derived independently from first principles -- and only afterward recognized as Chitragupta, implemented in code.
